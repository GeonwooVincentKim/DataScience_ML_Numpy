데이터 분석 프로세스
1. 기획
- 1. 문제 정의 -> 어떤 문제를 해결할 것인가
- 2. 분석 방법론 정의
  = 1. 기술 통계 분석
  = 2. 가설 검정
  = 3. 예측 모델 개발

2. 데이터 수집
- 1. 담당 부서로부터 데이터 입수
- 2. 데이터 탐색, 오류 검토

3. 분석
- 1. 데이터 탐색 및 전처리
- 2. 기술통계 분석 -> 집단 간 비교, 시계열 비교
- 3. 가설 검정 -> 가설 지지 확인
- 4. 예측 모형 개발, 고도화

4. 보고
- 1. 표, 그래프 제작
- 2. 보고서 작성 -> 문서화, 대시보드 개발


머신러닝이란
1. Model - 예측을 위한 수학 공식, 함수, 1차 방정식, 확률 분포, condition rule
2. Algorithm - 어떤 문제를 풀기 위한 과정, Model 을 생성하기 위한 훈련 과정
3. Feature
- 1. ML 에서 데이터의 특징을 나타내는 변수
- 2. feature, 독립변수, input 변수 등은 동일의미로 사용
- 3. 일반적으로 Table 상에 Data 를 표현 시, Column 을 의미
- 4. 하나의 data science (실제 Data) 는 feature vector 로 표현

4. continuous (연속형 값) - 값이 끊어지지 않고 연결됨, 온도, 시험평균 점수, 속도 등 일반적으로 실수 값들
5. discrete (이산형 값) - 값이 연속적이지 않음, 성별, 우편주소, 등수 등 Label 로 구분되는 값들
6. Numeric Types
- 1. 정량적으로 측정 가능한 data type
- 2. 일반적으로 정수 (integer) 또는 실수 (real-number) 로 표현
- 3. 온도, 자동차 속도, 날짜의 차이 (year or day)
- 4. 단위 (scale) 이 있는 Interval-scaled type
- 5. 비율이 있는 Ratio-scaled type

7. Nominal Types
- 1. Category 로 분류가 가능한 data type
- 2. 명목 척도라는 표헌으로 사용되기도 함
- 3. 색깔, 학교명, ID, 전공명 등
- 4. 두 개의 Category 만 분류 시, Binary Type 으로 구별

8. Ordinal Types
- 1. Category 로 분류가 가능하지만, 범주 간 순서가 있음
- 2. 명목 척도라는 표현으로 사용되기도 함
- 3. 음료수 병 크기, 학점, 5점 척도 설문조사
- 4. 측정되는 scale 또는 Unit 이 사람마다 다를 수 있음
- 5. 순서가 있는 것 != 배수로 증가하는 개념
